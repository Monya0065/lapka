Безопасность и соответствие (концептуально)

Демо-сайт показывает подходы к безопасной архитектуре. Это не юридическая консультация и не сертифицированное медицинское ПО.

Демо-сайт не хранит персональные данные.

Коротко о модели безопасности

1) Минимальные права доступа по ролям (RBAC).

2) Проверка и фильтрация AI-вывода перед показом пользователю.

3) План реагирования на инциденты и аудит критичных действий.

4) Концепция локализации и хранения данных в нужной юрисдикции.

RBAC

Доступ разделяется по ролям: Владелец, Врач, Администратор клиники. Каждая роль получает только необходимые функции (принцип минимальных привилегий).

Журналы аудита

Критичные действия фиксируются в неизменяемом журнале: вход, изменения записей, экспорт документов, админ-операции.

Шифрование

Передача данных: TLS в канале. Хранение: шифрование на уровне диска/БД и управление ключами с ротацией.

Резервные копии и хранение

Регулярные резервные копии, тест восстановления и политика хранения по категориям данных и срокам.

Реагирование на инциденты

Процедуры: обнаружение, классификация критичности, локализация, устранение, пост-мортем и уведомления по политике.

Локализация данных

Для РФ может потребоваться локализация персональных данных в РФ и отдельные трансграничные процедуры. Этот раздел носит концептуальный характер и не является юридической консультацией.

Архитектура AI-роутера (текстовая схема)

Вход -> Санитайзер входа (редакция PII)
      -> Policy Guard (блок диагноза/лекарств/доз)
      -> Model Router (выбор семейства по риску и задаче)
      -> Output Validator (regex + детерминированные правила безопасности)
      -> Fallback Handler (детерминированные/оффлайн-ответы)
      -> Observability (редактированные логи + метрики безопасности)

Правила валидации вывода

  • Блокировать названия лекарств по словарю: «drug vocabulary list in production».
  • Блокировать паттерны дозирования: mg/kg, ml/kg, слова «dose», «prescribe».
  • Блокировать фразы диагностической уверенности: «это точно», «скорее всего», «у вашего питомца».
  • Если правило нарушено — заменить ответ на безопасный шаблон эскалации в клинику.
